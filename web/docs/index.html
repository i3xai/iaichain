1. 安装 iai

推荐用官网安装脚本，从 GitHub Release 拉取当前系统预编译包。

一键安装（推荐）

$ curl -fsSL https://iaiaiai.ai/install.sh | bash

指定版本 / 目录

# 指定版本
$ curl -fsSL https://iaiaiai.ai/install.sh | bash -s -- --version v0.4.3

# 安装到自定义目录
$ curl -fsSL https://iaiaiai.ai/install.sh | bash -s -- --dir ~/.local/bin

验证

$ iai version
# 期望类似：iai-chain 0.4.3
当前 Release 若只有 macos-aarch64 资产，其他平台请用源码构建：cargo build --release。完整说明见仓库 START.md / USAGE.md。

2. 升级到最新包

已安装 iai 后，用内置命令从 GitHub Releases 拉取并替换当前二进制（无需再跑 install.sh）。

检查是否有新版本

$ iai upgrade check

升级到最新 Release

$ iai upgrade run
# 跳过确认：
$ iai upgrade run -y

升级到指定版本

$ iai upgrade run --to v0.4.3 -y

装完后执行 iai --version 确认。若本机用 systemd 跑 iai.service，默认会自动 restart；不想重启可加 --no-restart。

等价重装：curl -fsSL https://iaiaiai.ai/install.sh | bash。升级依赖 GitHub Release 已发布对应平台资产。

3. 启动：中继 + 队长 + 队员

本机开 3 个终端，模拟网络协调与两个独立节点。数据目录必须分开。

进程命令作用
中继iai relay --port 8790任务公告板、原子领取、入队状态（不记账）
队长IAI_HOME=~/.iai-cap IAI_RELAY=… iai serve --port 8787发任务、审批入队；控制台 :8787
队员IAI_HOME=~/.iai-mem IAI_RELAY=… iai serve --port 8788申请入队、申请角色；控制台 :8788
 
# 终端 0 · 中继
$ iai relay --port 8790

# 终端 1 · 队长
$ IAI_HOME=~/.iai-cap IAI_RELAY=http://127.0.0.1:8790 iai serve --port 8787

# 终端 2 · 队员
$ IAI_HOME=~/.iai-mem IAI_RELAY=http://127.0.0.1:8790 iai serve --port 8788
角色控制台
队长http://127.0.0.1:8787/console
队员http://127.0.0.1:8788/console

首次启动会生成控制台密码（看日志），或：

$ IAI_HOME=~/.iai-cap iai password show
$ IAI_HOME=~/.iai-mem iai password show

4. 入队审批（领取前置）

仅队长可发起任务。队员先申请入队（可填加入词）；队长可批准或填写原因拒绝（拒绝后本次不可再申请）。队长还可查看在线空闲队员并邀请。

  1. 设置角色（可选） 
    $ IAI_HOME=~/.iai-cap iai node role captain
$ IAI_HOME=~/.iai-mem iai node role member
  2. 查队长 node id 
    $ IAI_HOME=~/.iai-cap iai node status
    记下类似 captain.xxxx 的标识。
  3. 队员申请：队员控制台 →「团队」→「申请 / 审批 / 邀请」→ 填队长 id、角色与加入词 →「提交申请」。
  4. 队长批准/拒绝：队长同一面板 → 待审批列表 →「批准」或「拒绝」（须填原因）。

5. 队长创建任务

创建后任务进入招募中；网络可见任务会出现在中继公告板。招满后变为「招募完成可运行」，由队长启动执行。

  1. 队长控制台 →「+ 新建任务」（队员看不到新建入口）。
  2. 填写标题；仓库先点连通检测。
  3. 至少配置 1 个开发角色，招募数 ≥ 1。
  4. 可见性选 「网络可领取」（跨节点招募）或「仅本机」。奖金可先填 0。
  5. 创建成功后状态为 recruiting。

6. 队员申请角色 → 招满 → 队长启动

队员选择未招满角色申请（可填加入词），队长审批通过后正式占槽；全部招满后队长点「开始下发任务」。

  1. 队员控制台 → 市场页「网络任务公告板」→ 点角色旁 「申请加入」，可填加入词。
  2. 队长打开该任务详情 →「待审招募」→ 批准或拒绝（拒绝须填原因，本次不可再申请该角色）。
  3. 名额招满后状态变为 招募完成可运行；队长在任务详情点 「开始下发任务」。
队员申请入队 → 队长批准 → 队长新建任务（招募中）→ 队员申请角色 → 队长审批 → 招满可运行 → 队长启动
结果含义
已申请 / pending等待队长审批
招募完成可运行槽位已满，可启动
拒绝后不可再申请入队或该角色本次被拒
仅队长可发起/启动队员调用会 403
未配置中继跨节点时需 IAI_RELAY

7. 常见问题

command not found: iai

检查 PATH，或重新执行安装脚本；源码构建可用 ./target/release/iai。

两个节点数据串了

确认 IAI_HOME 分别是 ~/.iai-cap 与 ~/.iai-mem，不要都用默认 ~/.iai。

申请入队失败

两边都要带 IAI_RELAY=http://127.0.0.1:8790，且中继进程已启动。

安装脚本 404

官网入口：https://iaiaiai.ai/install.sh。也可：curl -fsSL https://raw.githubusercontent.com/i3xai/iaichain/main/install.sh | bash。
